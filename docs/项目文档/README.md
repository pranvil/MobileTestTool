## 🗺️ 文档使用指南

### 对于项目经理

1. 查看 [项目章程](./01-需求阶段/01-项目章程.md) 了解项目整体情况
2. 查看 [需求跟踪矩阵](./01-需求阶段/04-需求跟踪矩阵.md) 跟踪需求状态
3. 查看 [测试计划文档](./04-测试阶段/01-测试计划文档.md) 了解测试安排

### 对于开发人员

1. 首先阅读 [开发环境搭建指南](./03-开发阶段/01-开发环境搭建指南.md) 搭建环境
2. 阅读 [编码规范文档](./03-开发阶段/02-编码规范文档.md) 了解编码规范
3. 参考 [系统架构设计文档](./02-设计阶段/01-系统架构设计文档.md) 理解架构
4. 参考 [需求规格说明书](./01-需求阶段/02-需求规格说明书.md) 了解需求细节

### 对于测试人员

1. 阅读 [测试计划文档](./04-测试阶段/01-测试计划文档.md) 了解测试策略
2. 参考 [需求规格说明书](./01-需求阶段/02-需求规格说明书.md) 编写测试用例
3. 参考 [用户故事列表](./01-需求阶段/03-用户故事列表.md) 了解验收标准

### 对于新加入的团队成员

1. 阅读 [项目章程](./01-需求阶段/01-项目章程.md) 了解项目背景
2. 阅读 [开发环境搭建指南](./03-开发阶段/01-开发环境搭建指南.md) 搭建环境
3. 阅读 [编码规范文档](./03-开发阶段/02-编码规范文档.md) 了解开发规范

## 📝 文档维护

### 文档更新原则

1. **及时更新**：文档应该随项目进展及时更新
2. **版本控制**：文档应该有版本号和更新日期
3. **状态标识**：文档应该标明当前状态（草案/评审中/已批准）

### 文档评审流程

1. 作者编写文档
2. 提交评审
3. 相关人员评审
4. 根据反馈修改
5. 批准后发布
